# Hybrid Retrieval Results

Hybrid retrieval results should answer a simple question: does combining lexical and semantic retrieval actually improve the candidate set? If the answer is yes, the report should show this clearly with the same evaluation setting used for the single retrievers. If the answer is mixed, that is also a useful result and should be reported honestly.

At the moment, the main value of this subsection is that it gives a place to compare the modular fusion design against simpler retrieval-only baselines. That comparison is central to the project because the full system depends on the quality of the candidate set before reranking and generation.
