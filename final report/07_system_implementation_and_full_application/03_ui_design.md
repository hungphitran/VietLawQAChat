# UI Design

The UI should be designed to support both search and explanation. In a legal setting, a good interface does not only show the final answer; it should also show the evidence passages and the ranking order so that the user can inspect why the system answered in that way.

If the UI is still under development, the report can describe the target layout and the information hierarchy instead of pretending the interface is final. The most important design principle is clarity: the user should be able to see the question, the answer, and the supporting legal sources in one place.
